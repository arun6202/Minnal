## 3. The migration — `V001__initial_schema.sql`

This SQL is the source of truth for V001. It mirrors `Minnal.md §5.1` and adds **one new table** (`hook_executions`) that Minnal.md §5 alludes to in `Minnal-hooks` but does not define. Adding it now is cheaper than V002 immediately after V001.

```sql
-- V001__initial_schema.sql
-- Minnal initial schema. Append-only after first ship; never edit.
-- Foreign-key enforcement is enabled at connection-pool init, not here.

PRAGMA foreign_keys = ON;

-- --------------------------------------------------------------------
-- Schema version tracking
-- --------------------------------------------------------------------
CREATE TABLE schema_migrations (
    version     INTEGER PRIMARY KEY,
    applied_at  TEXT    NOT NULL,           -- RFC3339
    description TEXT    NOT NULL
);

-- --------------------------------------------------------------------
-- Collections
-- --------------------------------------------------------------------
CREATE TABLE collections (
    id          TEXT PRIMARY KEY,           -- UUID v7
    name        TEXT NOT NULL,
    sha         TEXT NOT NULL,              -- content-addressed (collection JSON SHA-256)
    git_path    TEXT,
    imported_at TEXT NOT NULL,              -- RFC3339
    updated_at  TEXT NOT NULL,              -- RFC3339
    source      TEXT NOT NULL CHECK (source IN ('postman', 'openapi', 'native'))
);
CREATE INDEX idx_collections_source ON collections(source);

-- --------------------------------------------------------------------
-- Requests
-- --------------------------------------------------------------------
CREATE TABLE requests (
    id              TEXT PRIMARY KEY,
    collection_id   TEXT NOT NULL REFERENCES collections(id) ON DELETE CASCADE,
    name            TEXT NOT NULL,
    method          TEXT NOT NULL CHECK (method IN
                        ('GET','POST','PUT','PATCH','DELETE','HEAD','OPTIONS','CONNECT','TRACE')),
    url_template    TEXT NOT NULL,
    headers         TEXT NOT NULL,          -- JSON: Vec<(String,String)>
    body_template   TEXT,                   -- nullable; JSON | raw | form-data sentinel
    auth_ref        TEXT REFERENCES auth_configs(id) ON DELETE SET NULL,
    schema_ref      TEXT REFERENCES contracts(id)    ON DELETE SET NULL,
    created_at      TEXT NOT NULL,
    updated_at      TEXT NOT NULL
);
CREATE INDEX idx_requests_collection ON requests(collection_id);

-- --------------------------------------------------------------------
-- Environments and variables
-- --------------------------------------------------------------------
CREATE TABLE environments (
    id    TEXT PRIMARY KEY,
    name  TEXT NOT NULL,
    scope TEXT NOT NULL CHECK (scope IN ('global','collection','request'))
);

CREATE TABLE variables (
    id              TEXT PRIMARY KEY,
    env_id          TEXT NOT NULL REFERENCES environments(id) ON DELETE CASCADE,
    key             TEXT NOT NULL,
    value_encrypted TEXT NOT NULL,          -- AES-256-GCM, key from Windows DPAPI
    source          TEXT NOT NULL CHECK (source IN ('manual','hook','ai_inferred')),
    UNIQUE (env_id, key)
);
CREATE INDEX idx_variables_env ON variables(env_id);

-- --------------------------------------------------------------------
-- Auth configurations
-- --------------------------------------------------------------------
CREATE TABLE auth_configs (
    id         TEXT PRIMARY KEY,
    scheme     TEXT NOT NULL CHECK (scheme IN
                  ('bearer','basic','oauth2_pkce','api_key','aws_sig_v4','mtls','custom')),
    config     TEXT NOT NULL,               -- JSON; sensitive fields encrypted (DPAPI)
    created_at TEXT NOT NULL
);

-- --------------------------------------------------------------------
-- Response bodies — content-addressed (ETL Parquet pattern)
-- Defined BEFORE responses so the FK from responses(body_sha) is resolvable.
-- --------------------------------------------------------------------
CREATE TABLE response_bodies (
    sha          TEXT PRIMARY KEY,           -- SHA-256 hex of raw body
    body         BLOB NOT NULL,              -- zstd-compressed
    content_type TEXT NOT NULL,
    stored_at    TEXT NOT NULL
);

-- --------------------------------------------------------------------
-- Response history
-- --------------------------------------------------------------------
CREATE TABLE responses (
    id          TEXT PRIMARY KEY,
    request_id  TEXT NOT NULL REFERENCES requests(id) ON DELETE CASCADE,
    timestamp   TEXT NOT NULL,
    status      INTEGER NOT NULL,
    latency_ms  INTEGER NOT NULL,
    headers     TEXT NOT NULL,               -- JSON
    body_sha    TEXT NOT NULL REFERENCES response_bodies(sha),
    size_bytes  INTEGER NOT NULL
);
CREATE INDEX idx_responses_request_time ON responses(request_id, timestamp DESC);

-- --------------------------------------------------------------------
-- Contracts (schema learning)
-- --------------------------------------------------------------------
CREATE TABLE contracts (
    id              TEXT PRIMARY KEY,
    request_id      TEXT NOT NULL REFERENCES requests(id) ON DELETE CASCADE,
    inferred_schema TEXT NOT NULL,           -- JSON Schema
    observed_at     TEXT NOT NULL,
    violation_count INTEGER NOT NULL DEFAULT 0,
    last_violation  TEXT
);
CREATE INDEX idx_contracts_request ON contracts(request_id);

-- --------------------------------------------------------------------
-- WASM hooks
-- --------------------------------------------------------------------
CREATE TABLE hooks (
    id            TEXT PRIMARY KEY,
    request_id    TEXT REFERENCES requests(id)    ON DELETE CASCADE,
    collection_id TEXT REFERENCES collections(id) ON DELETE CASCADE,
    phase         TEXT NOT NULL CHECK (phase IN ('pre','post')),
    wasm_sha      TEXT NOT NULL,             -- content-addressed WASM binary
    description   TEXT NOT NULL,             -- natural-language source of truth
    approved_at   TEXT,                      -- NULL = pending approval
    approved_by   TEXT CHECK (approved_by IN ('user','dry_run_passed') OR approved_by IS NULL),
    timeout_ms    INTEGER NOT NULL DEFAULT 5000,
    capabilities  TEXT NOT NULL,             -- JSON: HookCapabilities (see Minnal-hooks §6.3)
    CHECK (request_id IS NOT NULL OR collection_id IS NOT NULL)
);

-- Audit log: every hook execution recorded.
-- (Not in Minnal.md §5.1, but referenced by Minnal-hooks plan §5.4 M9; included
--  in V001 to avoid an immediate V002.)
CREATE TABLE hook_executions (
    id           TEXT PRIMARY KEY,
    hook_id      TEXT NOT NULL REFERENCES hooks(id) ON DELETE CASCADE,
    request_id   TEXT REFERENCES requests(id)       ON DELETE SET NULL,
    started_at   TEXT NOT NULL,
    duration_ms  INTEGER NOT NULL,
    outcome      TEXT NOT NULL CHECK (outcome IN
                    ('modified','passthrough','cancelled','timeout','error')),
    error_detail TEXT
);
CREATE INDEX idx_hook_executions_hook_time ON hook_executions(hook_id, started_at DESC);

-- --------------------------------------------------------------------
-- Assertions
-- --------------------------------------------------------------------
CREATE TABLE assertions (
    id         TEXT PRIMARY KEY,
    request_id TEXT NOT NULL REFERENCES requests(id) ON DELETE CASCADE,
    type       TEXT NOT NULL CHECK (type IN ('declarative','wasm')),
    spec       TEXT NOT NULL,                -- TOML (declarative) or wasm_sha
    created_at TEXT NOT NULL
);

-- --------------------------------------------------------------------
-- Dependency graph edges
-- --------------------------------------------------------------------
CREATE TABLE request_dependencies (
    from_request_id TEXT NOT NULL REFERENCES requests(id) ON DELETE CASCADE,
    to_request_id   TEXT NOT NULL REFERENCES requests(id) ON DELETE CASCADE,
    dependency_type TEXT NOT NULL CHECK (dependency_type IN
                       ('auth_token','response_field','ordering')),
    field_path      TEXT,
    inferred_by     TEXT NOT NULL CHECK (inferred_by IN ('ai','user')),
    PRIMARY KEY (from_request_id, to_request_id)
);

-- --------------------------------------------------------------------
-- Semantic embeddings (sqlite-vec virtual table)
-- ⚠️ Phase 0.5 Spike B confirms the embedding dimension before this ships.
--    If gemma-4-e4b emits a different dimension, change FLOAT[1024] HERE
--    before V001 ever leaves a developer's machine. After ship: append-only.
-- --------------------------------------------------------------------
CREATE VIRTUAL TABLE embeddings USING vec0(
    request_id TEXT,
    embedding  FLOAT[1024]
);

-- --------------------------------------------------------------------
-- Stamp this migration as applied
-- --------------------------------------------------------------------
INSERT INTO schema_migrations(version, applied_at, description)
VALUES (1, strftime('%Y-%m-%dT%H:%M:%fZ', 'now'), 'initial schema');
```

### 3.1 Schema deltas vs `Minnal.md §5.1` (intentional)

| Change | Why |
|---|---|
| `CHECK` constraints on every stringly-typed enum column | Defence-in-depth: even if a buggy writer bypasses the Rust enum, SQLite refuses bad data. |
| `response_bodies` defined *before* `responses` | FK resolution; SQLite doesn't allow forward references in `CREATE TABLE`. |
| `idx_responses_request_time` (request_id, timestamp DESC) | UC-04 needs "last 10 responses for this request" — this is the index that pays for it. |
| `idx_requests_collection`, `idx_variables_env`, `idx_contracts_request` | Hot lookup paths. Cheap to add now; expensive to add later under live data. |
| `hooks.capabilities` column added | Required by Minnal-hooks §6.3; absent from spec table. |
| `hook_executions` table added | Audit log (Minnal-hooks plan §5.4 M9). Avoids immediate V002. |
| `variables(env_id, key) UNIQUE` | Same key under same env is an upsert target, not two rows. |
| `ON DELETE CASCADE` on parent FKs (collections→requests etc.) | Prevents orphan rows. The data model has no concept of an orphan request. |
| `auth_configs.scheme` includes `'custom'` | Minnal.md §10 lists "Custom" as a v1 scheme; spec §5.1 omitted it. |

### 3.2 Schema deltas vs `Minnal.md §5.1` (open questions for Arun)

- **`response_bodies.body BLOB` zstd-compressed** — zstd level pinned at compile time? Suggested level 3 (good ratio, sub-millisecond on warm CPU).
- **`responses.headers TEXT` as JSON** — should this also be content-addressed? Same JSON object recurs across many responses (rate-limit headers, cache-control). Probably overkill for v1 — defer to v2 if profiling shows headers dominate disk.

## 4. Rust type layer

The whole point of this crate is that **no caller of Minnal-store ever holds a `String` that should have been a type**. This section is non-negotiable.

### 4.1 `ids.rs` — typed UUID-v7 newtypes

```rust
use uuid::Uuid;

macro_rules! id_newtype {
    ($name:ident) => {
        #[derive(Debug, Clone, Copy, PartialEq, Eq, Hash, serde::Serialize, serde::Deserialize)]
        #[serde(transparent)]
        pub struct $name(pub Uuid);

        impl $name {
            pub fn new() -> Self { Self(Uuid::now_v7()) }
            pub fn as_str(&self) -> String { self.0.to_string() }
        }

        impl ::sqlx::Type<::sqlx::Sqlite> for $name {
            fn type_info() -> ::sqlx::sqlite::SqliteTypeInfo {
                <String as ::sqlx::Type<::sqlx::Sqlite>>::type_info()
            }
        }
        // Encode/Decode via String round-trip (UUID → text in SQLite).
    };
}

id_newtype!(CollectionId);
id_newtype!(RequestId);
id_newtype!(EnvironmentId);
id_newtype!(VariableId);
id_newtype!(AuthConfigId);
id_newtype!(ResponseId);
id_newtype!(HookId);
id_newtype!(HookExecutionId);
id_newtype!(AssertionId);
id_newtype!(ContractId);
```

A `CollectionId` cannot be passed where a `RequestId` is wanted. The compiler stops the bug before it happens.

`BodySha` is *not* an ID — it's a content address. Defined separately:

```rust
#[derive(Debug, Clone, PartialEq, Eq, Hash, serde::Serialize, serde::Deserialize)]
#[serde(transparent)]
pub struct BodySha(pub String);          // 64-char lowercase hex SHA-256

impl BodySha {
    pub fn from_bytes(b: &[u8]) -> Self { /* sha2 + hex encode */ }
}
```

### 4.2 `enums.rs` — discriminated unions for every stringly-typed column

```rust

#[derive(Debug, Clone, Copy, PartialEq, Eq, Serialize, Deserialize)]

pub enum Method {

#[derive(Debug, Clone, Copy, PartialEq, Eq, Serialize, Deserialize)]

#[derive(Debug, Clone, Copy, PartialEq, Eq, Serialize, Deserialize)]
#[serde(rename_all = "snake_case")]
pub enum VariableSource { Manual, Hook, AiInferred }

#[derive(Debug, Clone, Copy, PartialEq, Eq, Serialize, Deserialize)]

// All implement TryFrom<&str> for tolerant decode (returns StoreError::EnumDecode).
```

The exact serialisation strings must match the SQL `CHECK` constraints in §3, byte-for-byte. Property test §6.2 asserts this round-trip.

### 4.3 `sealed.rs` — DPAPI-protected values

`variables.value_encrypted` and the sensitive fields in `auth_configs.config` must never be exposed as a plain `String`. The type system enforces this.

```rust
/// A value sealed by Windows DPAPI. Cannot be read without an Unsealer.
#[derive(Debug, Clone)]
pub struct Sealed<T>(Vec<u8>, std::marker::PhantomData<T>);

impl<T> Sealed<T> {
    pub fn from_ciphertext(bytes: Vec<u8>) -> Self { Self(bytes, Default::default()) }
    pub fn as_ciphertext(&self) -> &[u8] { &self.0 }
}

/// The Unsealer is the only path to plaintext. Acquired from the DPAPI provider
/// at session start; cannot be cloned or stored.
pub struct Unsealer { /* opaque DPAPI handle */ }
impl Unsealer {
    pub fn open<T: serde::de::DeserializeOwned>(&self, sealed: &Sealed<T>)
        -> Result<T, crate::StoreError>;
}
```

This blocks the most common leak: a future caller `let token: String = repo.get_variable(...)?;` that accidentally logs the token. With `Sealed<String>`, that line doesn't compile.

### 4.4 Schema structs — one per table

Each `schema/<table>.rs` defines a struct with `sqlx::FromRow`, with **typed columns** (no `String` where an enum or newtype belongs). Example:

```rust
// schema/request.rs
use crate::ids::*;
use crate::enums::*;
use sqlx::FromRow;
use time::OffsetDateTime;

#[derive(Debug, Clone, FromRow)]
pub struct Request {
    pub id:            RequestId,
    pub collection_id: CollectionId,
    pub name:          String,
    pub method:        Method,                          // not String
    pub url_template:  UrlTemplate,                     // newtype around String, validated
    pub headers:       HeaderList,                      // typed Vec<(HeaderName, HeaderValue)>
    pub body_template: Option<BodyTemplate>,            // discriminated: Json | Raw | FormData
    pub auth_ref:      Option<AuthConfigId>,
    pub schema_ref:    Option<ContractId>,
    pub created_at:    OffsetDateTime,
    pub updated_at:    OffsetDateTime,
}

#[derive(Debug, Clone)]
pub enum BodyTemplate {
    Json(serde_json::Value),
    Raw { content_type: String, bytes: Vec<u8> },
    FormData(Vec<FormField>),
}
// BodyTemplate (de)serialises to/from a tagged JSON form for storage.
```

## 5. Repository traits

One trait per aggregate root. Every method is `async`, every method returns `Result<_, StoreError>`. **Implementations are NOT in this design** — Codex scaffolds the trait + a `todo!()` impl; implementations come in M6 after Claude's design review.

```rust
// repo/collections.rs
#[async_trait::async_trait]
pub trait CollectionRepo: Send + Sync {
    async fn insert(&self, c: NewCollection) -> Result<CollectionId, StoreError>;
    async fn get(&self, id: CollectionId)    -> Result<Option<Collection>, StoreError>;
    async fn list(&self)                     -> Result<Vec<Collection>, StoreError>;
    async fn update(&self, c: Collection)    -> Result<(), StoreError>;
    async fn delete(&self, id: CollectionId) -> Result<(), StoreError>;
}

// repo/requests.rs
#[async_trait::async_trait]
pub trait RequestRepo: Send + Sync {
    async fn insert(&self, r: NewRequest)                        -> Result<RequestId, StoreError>;
    async fn get(&self, id: RequestId)                           -> Result<Option<Request>, StoreError>;
    async fn list_by_collection(&self, c: CollectionId)          -> Result<Vec<Request>, StoreError>;
    async fn update(&self, r: Request)                           -> Result<(), StoreError>;
    async fn delete(&self, id: RequestId)                        -> Result<(), StoreError>;
}

// repo/bodies.rs — content-addressed
#[async_trait::async_trait]
pub trait BodyStore: Send + Sync {
    /// Returns the SHA, whether newly inserted or already present (idempotent).
    async fn put(&self, body: &[u8], content_type: &str) -> Result<BodySha, StoreError>;
    async fn get(&self, sha: &BodySha)                   -> Result<Option<StoredBody>, StoreError>;
    /// Reference count check; never auto-deletes (see §7).
    async fn ref_count(&self, sha: &BodySha)             -> Result<u64, StoreError>;
}

// repo/responses.rs
#[async_trait::async_trait]
pub trait ResponseRepo: Send + Sync {
    async fn insert(&self, r: NewResponse)                            -> Result<ResponseId, StoreError>;
    async fn last_n_for_request(&self, req: RequestId, n: usize)      -> Result<Vec<Response>, StoreError>;
    async fn get(&self, id: ResponseId)                               -> Result<Option<Response>, StoreError>;
}

// repo/embeddings.rs — sqlite-vec OR hnsw_rs behind a trait
#[async_trait::async_trait]
pub trait EmbeddingIndex: Send + Sync {
    async fn upsert(&self, req: RequestId, vec: &[f32])               -> Result<(), StoreError>;
    async fn delete(&self, req: RequestId)                            -> Result<(), StoreError>;
    async fn search(&self, query: &[f32], k: usize)                   -> Result<Vec<(RequestId, f32)>, StoreError>;
}

// (Symmetric traits exist for environments, variables, auth_configs, hooks,
//  hook_executions, assertions, contracts, dependencies — same shape, omitted
//  for brevity; Codex generates them mechanically from the table list.)
```

`NewCollection`, `NewRequest`, etc. are insert-DTO structs — they omit `id`, `created_at`, `updated_at`, which the repo fills in. This makes "insert with a server-generated id" the only insert path.

## 6. Property tests (the acceptance gate, §5.1 M2 / M4 / M7)

These tests are not optional. PR fails if any are missing.

### 6.1 `content_addressed_dedupe.rs` — the ETL law

```text
For any non-empty body B and any positive N:
  putting B N times via BodyStore::put yields:
    - exactly N rows in `responses` (after corresponding insert_response calls)
    - exactly 1 row in `response_bodies`
    - sha is identical across all N puts
For any two distinct bodies B₁ ≠ B₂:
  put(B₁) and put(B₂) yield distinct shas.
```

Implementation: `proptest!` with strategy `prop::collection::vec(any::<u8>(), 1..4096)`.

### 6.2 `enum_round_trip.rs` — defence-in-depth

```text
For every enum variant V of every discriminated-union enum:
  insert a row with V, select it back, assert variant equality.
For every CHECK-rejected string S (e.g. "GET " with trailing space):
  insert is rejected with StoreError::Sqlx (constraint failure).
```

### 6.3 `uuid_v7_monotonic.rs`

```text
For any sequence of N IDs minted within a single thread:
  the lexicographic order of their string forms is non-decreasing.
```

This is the property that makes `ORDER BY id` equivalent to `ORDER BY created_at` for our common queries.

### 6.4 `variable_scope_resolution.rs`

```text
Given a request R in collection C with environments at all three scopes
defining key K with values v_global, v_collection, v_request:
  the resolver returns v_request.
Removing the request-scope value:
  the resolver returns v_collection.
Removing the collection-scope value too:
  the resolver returns v_global.
With no value defined at any scope:
  the resolver returns None (not "" — None).
```

Note: the resolver itself lives in **Minnal-core**, not Minnal-store. The store's job is to expose the data correctly; the property test against the store verifies that scope and key are preserved on round-trip without conflation.

## 7. Garbage collection — what V001 does NOT do

**Body GC is deferred to v1.1.** Rationale:
- A response body could be referenced by a stored response, by a contract example, by a future export bundle, by a v2 dependency-graph snapshot. Building the reference graph correctly is a v1.1 problem.
- Disk is cheap; correctness is not. Until ref-graph is bulletproof, **never delete a row from `response_bodies`**.
- The `ref_count(sha)` method exists for inspection / future GC, not for active deletion.

If Codex is tempted to write a `cleanup_unreferenced_bodies()` helper: don't. PR fail.

## 8. Connection-pool init — the things easy to forget

Every `SqlitePool` must be created with these `PRAGMA`s applied per-connection:

```rust
PRAGMA foreign_keys = ON;            -- not on by default in SQLite
PRAGMA journal_mode = WAL;           -- concurrent reads with writers
PRAGMA synchronous = NORMAL;         -- WAL + NORMAL is the standard durable+fast combo
PRAGMA busy_timeout = 5000;          -- never panic on momentary lock contention
PRAGMA temp_store = MEMORY;
```

Plus, when sqlite-vec is in use, `vec0` must be loaded:

```rust
sqlite_vec::load(&conn)?;            // before any embeddings query
```

Capture this in `lib.rs::open(path)` so callers can never forget.

## 9. The sqlite-vec / hnsw_rs branch (per plan.md §5.1 M7)

Both implement the `EmbeddingIndex` trait from §5. The choice is made at compile time via a Cargo feature:

```toml
[features]
default = ["vec-sqlite-vec"]
vec-sqlite-vec = ["dep:sqlite-vec"]
vec-hnsw       = ["dep:hnsw_rs"]
```

If Phase 0.5 sqlite-vec spike fails on Windows, we flip the default. **No call site changes.** This is the whole point of having the trait.

The HNSW path stores embeddings as a `BLOB` column in a regular table (added to V001 if HNSW wins, omitted if sqlite-vec wins — branch resolved before V001 ships).

## 10. Things Codex must NOT silently change

These are design decisions, not stylistic choices:

1. **Foreign-key enforcement at pool init.** Not in V001 (PRAGMA is per-connection, not per-database).
2. **`hook_executions` table in V001, not V002.** Saves an immediate follow-up migration.
3. **`Sealed<T>` newtype** — variables and auth secrets must not be plain `String`.
4. **`EmbeddingIndex` trait** — both backends implement it; no direct sqlite-vec calls outside `vec_index.rs`.
5. **`NewX` insert-DTOs** — id and timestamps are repo-generated, not caller-supplied.
6. **No `unwrap()` outside `#[cfg(test)]`.** This is the review gate.
7. **No `String` for enum-shaped values in any public type.**

If any of the above must change, open a design-review issue first; do not change-then-ask.
